📋 Production Systems & Rule-Based Reasoning

The Foundation of Symbolic AI Reasoning Engines

Learning Objectives:
Understand how production systems work as a reasoning engine, learn to design IF-THEN rules, master pattern matching and conflict resolution, and build practical rule-based systems for decision-making.

Introduction: What Are Production Systems?

A production system is one of the oldest and most powerful concepts in AI reasoning. It's elegantly simple: a set of IF-THEN rules operating over a working memory, where the system continuously checks which rules apply and executes matching actions.

Core Idea:
A production system operates in a simple loop:
  1. Pattern Matching: Check which rules' conditions match current memory state
  2. Conflict Resolution: If multiple rules match, decide which one fires first
  3. Action Execution: Execute the rule's action (modify memory or act in world)
  4. Repeat: Continue until no more rules can fire

This might seem simple, but production systems are Turing-complete—they can compute any computable function. And unlike neural networks, every decision is traceable: you can inspect which rules fired and why.

Why Production Systems Matter for Neuro-Symbolic AI

Production systems provide the symbolic reasoning backbone for neuro-symbolic AI

Anatomy of a Production System

Component Description
Rules IF (condition) THEN (action) statements. Conditions are patterns to match against memory.
Working Memory Current state of facts/data. Rules match against this. When rules fire, memory updates.
Pattern Matcher Engine that checks which rules' conditions are satisfied by current memory.
Conflict Resolver When multiple rules match, decides which one fires next (specificity, recency, priority).
Inference Engine The execution loop: match → resolve → fire → update → repeat.

Pattern Matching: The Heart of Production Systems

Pattern matching is how the system determines which rules apply. A rule's condition is a pattern that must match facts in working memory.

Simple Medical Diagnosis Example

Working Memory (current facts):

patient:
  age: 25
  symptom_fever: true
  symptom_cough: true
  symptom_sore_throat: true
  duration_days: 3
  severity: mild

Rules:

RULE 1: "Viral URI"
IF
  symptom_fever = true
  AND symptom_cough = true
  AND severity != severe
  AND duration_days < 7
THEN
  diagnosis = "Viral Upper Respiratory Infection"
  treatment = "Rest, fluids, supportive care"

RULE 2: "Bacterial Infection"
IF
  symptom_sore_throat = true
  AND symptom_fever = true
  AND severity = severe
THEN
  diagnosis = "Bacterial Pharyngitis"
  treatment = "Antibiotics required"

RULE 3: "Common Cold"
IF
  symptom_cough = true
  AND duration_days < 5
  AND fever = false
THEN
  diagnosis = "Common Cold"
  treatment = "Time and TLC"

Pattern Matching Result:

Checking RULE 1 against memory:
  symptom_fever = true ✓
  symptom_cough = true ✓
  severity != severe (mild != severe) ✓
  duration_days < 7 (3 < 7) ✓
→ RULE 1 MATCHES!

Checking RULE 2:
  symptom_sore_throat = true ✓
  symptom_fever = true ✓
  severity = severe (but memory says 'mild') ✗
→ RULE 2 DOES NOT MATCH

Checking RULE 3:
  symptom_cough = true ✓
  duration_days < 5 (3 < 5) ✓
  fever = false (but memory says true) ✗
→ RULE 3 DOES NOT MATCH

Result: Only RULE 1 matches. It fires and produces the diagnosis "Viral URI".

Conflict Resolution: When Multiple Rules Match

What happens when multiple rules match? The conflict resolver decides which rule fires next. Common strategies:

Strategy How It Works Pros Cons
Specificity Fire the rule with most conditions (most specific match) Intuitive, handles edge cases well Can be unpredictable if not careful
Recency Fire the rule that matches the newest facts Responds to recent events May miss important standing rules
Priority Rules have explicit priority numbers (1=highest) Explicit, easy to understand and audit Requires careful design
Lex (Left-to-Right) Fire the first rule that matches (in order) Simplest, deterministic Order-dependent, fragile to reordering

Conflict Resolution in Medical Diagnosis

Imagine a patient with fever, sore throat, and cough. Multiple rules match:

RULE A (3 conditions matched):
  fever AND cough AND sore_throat
  → Diagnosis: "Viral Pharyngitis"

RULE B (2 conditions matched):
  fever AND cough
  → Diagnosis: "Generic Flu"

RULE C (1 condition matched):
  fever
  → Diagnosis: "Fever - investigate further"

Using SPECIFICITY strategy:
→ RULE A fires (3 conditions = most specific)
→ Output: "Viral Pharyngitis"

Using PRIORITY strategy (if defined):
RULE A: priority 10
RULE B: priority 20
RULE C: priority 30
→ RULE A fires (10 < 20, so highest priority)
→ Output: "Viral Pharyngitis"

Building Production Systems in Practice

# Simple Production System in Python

class ProductionSystem:
    def __init__(self):
        self.working_memory = {}
        self.rules = []
        self.fired_rules = []

    def add_rule(self, name, conditions, actions, priority=0):
        """Add a rule to the system"""
        self.rules.append({
            'name': name,
            'conditions': conditions,  # List of condition functions
            'actions': actions,        # List of action functions
            'priority': priority
        })

    def pattern_match(self):
        """Find all rules whose conditions are satisfied"""
        matching_rules = []
        for rule in self.rules:
            # Check if all conditions are true
            all_satisfied = all(
                condition(self.working_memory)
                for condition in rule['conditions']
            )
            if all_satisfied:
                matching_rules.append(rule)
        return matching_rules

    def resolve_conflict(self, matching_rules):
        """Choose which rule to fire (by priority)"""
        if not matching_rules:
            return None
        # Sort by priority (lower number = higher priority)
        return min(matching_rules, key=lambda r: r['priority'])

    def fire_rule(self, rule):
        """Execute a rule's actions"""
        for action in rule['actions']:
            action(self.working_memory)
        self.fired_rules.append(rule['name'])

    def run(self, max_iterations=100):
        """Run the production system until no more rules can fire"""
        iteration = 0
        while iteration < max_iterations:
            matching = self.pattern_match()
            if not matching:
                break  # No more rules to fire

            rule_to_fire = self.resolve_conflict(matching)
            self.fire_rule(rule_to_fire)
            iteration += 1

        return self.fired_rules


# Example: Medical Triage System
triage = ProductionSystem()

# Add rules
triage.add_rule(
    "red_flag_check",
    conditions=[
        lambda mem: mem.get('symptom_difficulty_breathing') == True,
        lambda mem: mem.get('severity') in ['severe', 'critical']
    ],
    actions=[
        lambda mem: mem.update({'recommendation': 'GO TO ER IMMEDIATELY'})
    ],
    priority=1  # Highest priority
)

triage.add_rule(
    "viral_uri",
    conditions=[
        lambda mem: mem.get('symptom_fever') == True,
        lambda mem: mem.get('symptom_cough') == True,
        lambda mem: mem.get('severity') in ['mild', 'moderate']
    ],
    actions=[
        lambda mem: mem.update({'recommendation': 'Rest, fluids, monitor'})
    ],
    priority=5
)

# Set initial working memory (patient data)
triage.working_memory = {
    'symptom_difficulty_breathing': False,
    'symptom_fever': True,
    'symptom_cough': True,
    'severity': 'mild'
}

# Run the system
triage.run()
print(f"Fired rules: {triage.fired_rules}")
print(f"Recommendation: {triage.working_memory.get('recommendation')}")

Key Takeaways

  • Production systems are transparent rule engines: every decision is traceable
  • Pattern matching determines which rules apply; rules fire when conditions match
  • Conflict resolution handles multiple matching rules—use priority, specificity, or recency
  • In neuro-symbolic AI, production systems follow neural perception to enforce logic
  • Auditable and safe: hard constraints can be enforced where neural models are uncertain
